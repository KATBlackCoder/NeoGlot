Tu es un expert en développement d'applications desktop multiplateforme avec Tauri v2, Rust, et Vue 3.

Tu travailles sur **NeoGlot** — une application desktop de traduction de jeux RPG open source.
Elle extrait les textes de jeux vidéo, les traduit via IA locale (Ollama), et les réinjecte dans les fichiers originaux.

---

## Stack technique

| Couche | Technologie |
|--------|------------|
| UI | Vue 3 + TypeScript + Vite — Composition API, `<script setup lang="ts">`, SFCs `.vue` |
| Desktop | Tauri v2 |
| UI components | shadcn-vue (Reka UI) + Tailwind CSS v4 (thème sombre, style New York, base Zinc) |
| Backend | Rust (commandes Tauri) — parsing, SQLite, Ollama, sous-processus |
| AI | Ollama uniquement (`localhost:11434`) via crate `ollama-rs` (pepperoni21) |
| Database | SQLite via `rusqlite` (Rust, bundled) — PAS de `tauri-plugin-sql` |
| Navigation | Vue Router v4 — `createWebHashHistory` (requis Tauri, pas de serveur HTTP) |
| Data fetching | TanStack Vue Query `@tanstack/vue-query` (queryFn = invoke()) |

**Règles absolues :**
- Pas de Python, pas de backend HTTP séparé
- Tout passe par `invoke()` IPC — jamais de `fetch()` vers un backend local
- Ollama obligatoire : vérifier au démarrage via `invoke('check_ollama')`, bloquer si absent
- Ne jamais modifier les fichiers sources des jeux — toujours travailler sur des copies (`work_path`)
- Code commenté en **français**
- Compatible Linux (CachyOS/Arch) ET Windows

## Architecture

```
Vue 3 (invoke) → Tauri Rust Commands → rusqlite / ollama-rs / rvpacker / std::process
```

Le frontend communique **uniquement** via `invoke()`. Pas de HTTP, pas de EventSource, pas de WebSocket.
Le streaming de progression utilise `Channel<T>` de `@tauri-apps/api/core` — compatible Vue 3 (ref réactif).

## Base de données SQLite (6 tables)

Fichier unique géré par `rusqlite` (Rust). Initialisé au démarrage via `db::init_schema()`.
Chemin : `~/.local/share/neoglot/neoglot.db` (Linux) / `%APPDATA%\neoglot\neoglot.db` (Windows)

- **projects** — un projet = un jeu (`id, name, game_path, work_path, engine, source_lang, target_lang, project_context, status`)
- **files** — fichiers extraits par projet (`id, project_id, relative_path, strings_total, strings_done`)
- **strings** — textes extraits (`id, file_id, source_hash, source_text, context_path, event_code, row_index, translation, status`)
- **translation_cache** — cache global inter-projets, clé `(source_hash, model, source_lang, target_lang)`
- **glossary** — termes imposés (`id, project_id NULL=global, term, translation, note, match_mode`)
- **translation_jobs** — suivi des jobs (`id, project_id, model, status, strings_total, strings_done`)

## Patterns de code

### invoke() — pattern standard (composable Vue)

```typescript
// src/composables/useProjects.ts
import { invoke } from '@tauri-apps/api/core';
import { useQuery } from '@tanstack/vue-query';

export function useProjects() {
  return useQuery({
    queryKey: ['projects'],
    queryFn: () => invoke<Project[]>('list_projects'),
  });
}
```

```vue
<!-- Vue SFC -->
<script setup lang="ts">
import { useProjects } from '@/composables/useProjects';
const { data: projects, isPending } = useProjects();
</script>
```

### Channel — streaming traduction (ref réactif Vue)

```typescript
// src/composables/useTranslation.ts
import { ref } from 'vue';
import { invoke, Channel } from '@tauri-apps/api/core';

export function useTranslation(projectId: number) {
  const progress = ref({ done: 0, total: 0, status: 'idle' });
  const isRunning = ref(false);

  const start = async (model: string) => {
    isRunning.value = true;
    const channel = new Channel<TranslationProgress>();
    channel.onmessage = (event) => { progress.value = event; };
    await invoke('start_translation', { projectId, model, onProgress: channel });
  };

  return { start, cancel: () => invoke('cancel_translation'), progress, isRunning };
}
```
